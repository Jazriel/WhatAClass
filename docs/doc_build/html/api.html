API¶

This part of the documentation covers the interfaces of WhatAClass.

WhatAClass¶

Simple web app designed to be the GUI of another app. Provides users a simple way to upload files and request results. Repository

WhatAClass.controllers¶

The usual web-app architecture is the MVC (Model-View-Controller). This class implements the controllers of the architecture.

The structure used was Blueprints for the app to be scalable (create a divisional or functional structure and distribute the work among the contributors).

author:Javier Martínez
WhatAClass.controllers.base()[source]¶

Controller for the index view.

WhatAClass.controllers.confirm_email(token)[source]¶

Try to see if the token is actually de-cypher-able and try to change the user to confirm the email.

WhatAClass.controllers.login()[source]¶

Try to log in the user with the information provided.

WhatAClass.controllers.logout()[source]¶

Logs the user out, has no effect if there was no one logged in.

WhatAClass.controllers.recover(token)[source]¶

Try to get the email from the token and give the chance to change password.

WhatAClass.controllers.reset()[source]¶

Reset the password given through an email with a time sensitive link.

WhatAClass.controllers.sign_up()[source]¶

Creates a user from the email and password given and sends an email with a time sensitive serialized url to authenticate the user (ts).

WhatAClass.models¶

The usual web-app architecture is the MVC (Model-View-Controller). This class implements the models of the architecture.

author:Javier Martínez
class WhatAClass.models.User(email, password, email_confirmed=False)[source]¶

User class, implemented with the help of SQLalchemy to be persistent. Basically has a id and an email both unique, a password, a field to know if the password has been confirmed and the is_active field needed by flask.
